Many people consider Microsoft to be this all-powerful entity that always gets its way and succeeds in whatever it attempts (whether through technical ability, luck, marketing, or illegal means). But in fact, if you look back at all the initiatives Microsoft has attempted, most of them have failed. Remember Windows at Work, an attempt to unify all office machines on a network? Or TrueImage, a clone of the PostScript page description language? Or ACE, the Advanced Computing Environment, designed to replace the Intel PC architecture? If you look at all the things that Microsoft has boldly announced with a gaggle of industry partners, the success rate is maybe 10 to 20 percent. So pick any ten things that Microsoft is attempting now: Auto PC, cell phone operating system, set-top box, Xbox, electronic books, Internet Gaming Zone, Tablet PC, Pocket PC, speech recognition, MSN. My view: Simply put, I believe that Microsoft is a vicious competitor. Do I believe that the actions it has taken to compete are unfair? Yes, but I don't think that it came from specific management strategies, but more from a mentality to compete. From what I read Microsoft breeds a brutal competitive atmosphere and allows too much independence to it's various departments. Perhaps Microsoft is just too good a competitor and needs to let it's enemies catch up. Microsoft needs to realize that, even though it is a very unstable market were a good idea can overtake a giant's, it still can't afford to act like it's playing catch up and act like the leader it is.

Yes, I still want to believe in Microsoft.

All page formatting is done in DOM, so in other words the sidebar is just a webpage. CSS and scripting all apply the same as if it were in the main window. So, CSS is how I chose to apply style. target="_content" in the "a" tags makes the links display in the main window instead of the side bar. Sidebar information can be found here and an AmphetaDesk template explanation is here. (Don't know perl? Using MoveableType with DHTML Javascript

As if you didn't know, MoveableType is the BOMB! Well, OK, maybe not that good. But it's what I use for my blog engine and it makes things much easier. As with most blog software that uses the BloogerAPI; it uses a template system to generate the webpage and manage content. Allowing you to focus on your sites news and articles instead of formatting each and every page. MoveableTypes system uses a proprietary system a little different than most. I like to think that it's a little bit easier to use and more flexible. The only problem is getting to know tag system and how it works. Using this system is really easy once you know the lingo, but how do you use the DHTML script to get that ubercool navigation bar?

Here it is in all it's glory.


<MTCategories>

<div id="cat<$MTCategoryID$>">

<div class="sidetitle">

<a href="javascript:catList('cat<$MTCategoryID$>')">[+]</a> <a href="<MTCategoryArchiveLink>"><$MTCategoryLabel$></a>

</div>

<div class="catlist">

<div class="side">

<MTEntries lastn="5">

<a href="<$MTEntryLink$>"><$MTEntryTitle$></a><br />

</MTEntries>

</div>

</div>

</div>

</MTCategories>


Lovely isn't it. It probably provides a little more perspective on why I wrote the script the way I did. Notice the catlist class selector. That selector is the one that gets created in memory when the script loads. It doesn't reside in my CSS file. "Side" and "Sidetitle" are selectors in my CSS file. They actually control the style even when the script isn't running, provided your browser supports CSS. The granularity built in to this sytem makes for a very general way to control the visibilty of you sheet without recoding everytime you add or delete categories or otherwise change your navbar. Also of note is the way "catlist" holds the same value of the javascript perimeter allowing the script to find the correct "catlist" to hide or show.

Nevertheless, this will display all your categories you have in your blog along with the last 5 entries in each. Well, that's about it for MT tags. Enjoy your new navigation bar.

DHTML via Javascript - A Collapsable Text Menu Tutorial

I've been trying to provide as much functionality to this site without making things too complicated for the user. As you can see the navigation bar on the main page has been updated. Clicking the "[+]" will expand the category now. I didn't like the long list of recent articles running down the page, but I wanted to provide a way to get a quick look at the various topics. Let me know what you think.

Now you know what I've been doing for the past couple of days. That's right, I learned to program Javascript. I've never written a program before three days ago. It was hell, but I fully intend to make your's no where as hard as it was for me. Given the plethora of programming centric sites out there, you would think that it would be a simple deal to learn. At the very least, you should be able to get a few clear examples. And in fact you can, if you want to create useless programs that do nothing. The problem is that most sites either assume that you already now how to program in another language or that you know nothing about it. The latter being the worst because they want to walk you through that process step by step while you write junk programs meanwhile totally skirting the real concepts that you need to know.

We won't do that here. If you're interested we'll delve right in.

Writing a program is all about moving and changing objects. it's really simple and frustrating at the same time once you realize that fact. Other places you will read about object oriented programming and how it's like "the way" to program. Well yeah it is, but for a beginner, you will mostly need to concern your self with much lower level objects. I won't do the standard "what is an object" talk, but I will tell you it's all about the API. That is Application Programming Interface. I'll be doing most of the grunt work here and indeed most every program nowadays.

For clarification, we'll only be talking about Javascript this time around. Why Javascript? Because it the easiest lang. that I've found yet. (And the most widespread for its use in DHTML and other web technologies) The most important thing you will need to know before you can begin is a few resources:

Netscape DevEdge - Good reference and documentation site.
W3C DOM - The W3C's Document Object Model Specification. It explains all the types and inheritance and interface properties.

You have to do lots of research to understand what each of the DHTM interfaces do and how to use them. This is crucial to understanding what you are moving around inside the containers. But with out further ado, here is the full script used to control the navigation bar.


function catList(id){
a="+";
b="-";
i=document.getElementById(id);
dval=i.getElementsByTagName("div").item(1).style.display;
t=i.getElementsByTagName("div").item(0).getElementsByTagName("a").item(0).firstChild;
if (dval=="block") {
dval="none";
t.nodeValue=t.nodeValue.replace(b,a);
}
else {
dval="block";
t.nodeValue=t.nodeValue.replace(a,b);
}
i.getElementsByTagName("div").item(1).style.display=dval;
}
if (document.getElementById && document.createTextNode){
document.write('<style type="text/css">.catlist {display:none;}</style>')
}

Purdy ain't it! I still can't believe I spent about 40 hours on this alone. Good thing is, it's reusable otherwise I would have to shoot myself.

I only used W3C DOM specification interfaces to the script should work with any DOM2 browser. IE 5.5 and Netscape 4 qualify for most funtions and Mozilla is fuly qualified. Note:My implementation does not qualify as XHTML. I've used the tag to insert the Category name to provide a unique id for each instance. It doesn't qualify because the names have spaces and CSS id specification doesn't allow spaces in the id. I'll have to find a work around, but this will do for now. You can alternatively use a manual naming format. UPDATE: Although at this time the validation site is down, it should now validate. Dummy me, I forgot about the tag. It's numeric and provided the unique identifier that was needed. I'll post the code in a separate article. It's pretty nifty.

Let the picking begin; starting with the top:


function catList(id){

"function" - we use this to establish a command to call from the web page, i.e. from a link or button.
"catList" - This is the name of the function, I could have named it catNip for that matter.
"(id)" - This is part of the "catList" function. "id" is a variable used to transfer a value to an internal function. (See below) I use this to specify the collection of tags that I want to modify. On the navigation bar it's the category name.
Of course this is were those other tutorials come in handy, syntax. We need a bracket "{" to contain the function. Otherwise you get *gasp* a syntax error. (Whatever shall we do?)

a="+";
b="-";

"a" - is a variable the equal (=) sign assigns the right hand value to the left hand side. Notice I wrote that before I used it in an expression later. Variables must be declared before they are used. "+" is going to be one of the characters is replace via a later expression.
"b" - the other value that I replace with "a".

i=document.getElementById(id);

"i" - another variable that I use to transfer the contents to another container.
"document" - Hey, it one of the those interfaces in our API defined by the DOM. This container performs a generic operation. It finds and returns the current page in the browser window as an object. We use any function or parameter that can be access by the "document" interface. (See what I mean by research. Check out the W3C site for more information.)
"getElementById" - Finds and returns all tags (elements) with the #id matching our parameter "id". No quotes in () because this is a variable being specified. This one was specified earlier in the "function" parameter. You can use any text value for the function which is not always true as we sill see later.
";" - used to delaminate the end of a line. More of that syntax stuff.

dval=i.getElementsByTagName("div").item(1).style.display;

"dval" - variable to hold the value for our CSS style being defined.
"i" - used to limit the expression to the contents of the specified ID.
"getElementsByTagName("div")" - another contained used to filter by a certain tag, in this case the "div" tag. This container returns a list of tags, not just a generic object.
"div" - The HTML tag to filter for.
"item(1)" - Index value to filer for in the list. Note: counting starts with 0 so in this case we will return the second tag in the list, by specifying "1".
"style" - Command used to perform a style definition on the object (tag).
"display" - type of styling that we want to do on this tag. This could be any CSS style definition.

t=i.getElementsByTagName("div").item(0).getElementsByTagName("a").item(0).firstChild;

"t" - Varible need I say more.
"i" - You should know what this is by now.
"getElementsByTagName("div")" - See how easy this is.
"item(0)" - Return the first "div" tag.
"getElementsByTagName("a") - Return a list of "a" tags.
"item(0)" - Return first "a" tag.
"firstChild" - DOM interface return the first object matching our criteria. (Later it is defied by either the "+" or "-" in the varibles "a" or "b".)

if (dval=="block") {
dval="none";
t.nodeValue=t.nodeValue.replace(b,a);
}

"if" - conditional statement that executes the contents only if the parameters are "true".
"(dval=="block")" - Parameter that returns true if the variable dval and "block" are equal. The "==" delaminate a true or false statement if both sides are equal.
"dval="none"" - sets the value of dval to "none" if this container is executed.
"t" - Umm, yeah....
"nodeValue" - returns the value as text instead of an [object]. Allows us to evaluate and execute the rest of the string.
"replace" - replaces the text per the parameters specified. This were we actually use those values for "a" and "b".

else {
dval="block";
t.nodeValue=t.nodeValue.replace(a,b);
}

"else" - executes if no other conditiond have be met. This statement sets the value of dval to "block". It also replaces the text of "a" with "b".

i.getElementsByTagName("div").item(1).style.display=dval;

"i.getElementsByTagName("div").item(1).style.display=dval;" - Recursive statement. It makes a loop.

if (document.getElementById && document.createTextNode){
document.write('<style type="text/css">.catlist {display:none;}</style>')
}

Simply, this writes a style sheet in memory, to allow for a more responsive click. "catlist" is an CSS class selector specified in my webpage. It defaults the value of display to "none"; hiding the menu by default. Added side effect is if the script doesn't execute for whatever reason, the display will NOT hide the contents of the class. Therefore the category menu will always show the entries if the script is not run, a neat side effect for those older browsers.

So lets look at the whole script. You click the hyperlink, the script looks at the document, finds the CSS id specified by the text in the hyperlink via the "id" variable, finds the second "div" tag, then sets the CSS display value for that CSS ID selector and sets it to hide or show the contents of the "catlist" class selector.
At the same time, it finds the document, the CSS id specified, the first "div" tag therein, the first "a" tag, the first character, and replaces the "+" sign with "-".

So, there you have it. Copy it, paste it, do whatever you like. Fell free to *gasp* change it.

I hope that was a bit more refreshing than the "other" programming tutorials on the web. If you still are having problems remember look in the documentation. The information there is much more useful and free of bloated oversimplified examples. Make sure you watch your syntax, watch for inheritance problems (if it doesn't explicitly say you can do something, most likely you can't without substantial reworking), and don't forget declare your variables before you use them.

And You Thought Ubiquitous Computing Was Dead

Anyone reading computer news about 2 years ago was bound to have read something about "ubiquitous computing" or "pervasive computing". I can't say that I was sad to stop hearing about it in the news. It was everywhere; everyone was so excited about it and what it was going to do for computers. A new paradigm they said. Well, finally, after many months, the fad faded away and settled into mediocrity. A bygone era of naive optimism.

I'm supposed to be wearing an internet enabled pair of shoes right now, and my jacket lining is supposed to be a display. Were did it all go? What happened, surely this wasn't just another hopeless expression of geek-so-shiek technolust. Did everything have to go with the dot-com bust?

Most assuredly not! It simply faded into the background where it was supposed to be in the first place. Slowly and quietly web services were rolled out to facilitate everyday functions such as news reading (XML::RDF::RSS). Along with a whole slew of other Web API's to facilitate application integration with the web and further contribute to the ideal that is the semantic web. The hardware is slowly becoming portable, powerful, and efficient enough to allow better integration.

But where does that leave us? Why haven't we seen more of this technology? It leaves us right where we are right now. Obiquitous computing never promised sexy new gadgets or technologies. Cut though all the hype that was so prevalent and you see a new picture. One where you don't see computers and yet they still work for you. One that doesn't require a BS degree in computer science to operate. That was the promise.

With that said; I don't think we have arrived, but we have indeed started the journey. We have much to learn on the way. But the infrastructure is being built. Wireless networks (wLAN), via 802.11, are become more common even in daily life. We'll soon have "personal area networks" (PAN), via bluetooth, and with that we have constant connectivity.

Top Sytle Pro 3 Review

Well, I promised it to you and, finally, after long debate and many revisions, I present my TopStyle Pro HTML Editor, CSS Editor, XHTML Editor for Windows review. For those wondering, Top style provides a intuitive interface and handy tools to validate and clean up what would otherwise be very tedious and time consuming work.

This is just the first of the original content that I plan to provide on this stie. My plans include much more. To include: detailed reviews, witty essays, and powerful editorials. Along with my daily web commentary, and insiteful news post. Provided I have the time of course. This is just a hobby for me after all.
I found Top Style Pro 3 from a need to edit my CSS files, but what I found was so much more. I can't say adaquately how much I love this program. Enthusiasm aside, let's dig in.
No really, what is TS. Feature list is as follows, ripped right from the web site:

HTML and XHTML editing - TopStyle comes into its own as a powerful HTML and XHTML editor!
Style Upgrade - Upgrade your HTML documents by replacing outdated markup with equivalent styling. Convert all of your font tags to CSS with a single click!
HTML Tidy - Validate and reformat your HTML with TopStyle's new HTML Tidy integration. Includes a Tidy configuration that converts your HTML to XHTML with no effort!
Editor Hyperlinking - TopStyle Pro 3.0's unique hyperlinking greatly simplifies navigating between documents.
Full Screen Preview - Display your HTML or CSS documents in a full screen preview which can be split between Internet Explorer and Mozilla (Netscape Gecko).
Related Styles - See at a glance which styles are related to the current HTML tag or CSS selector.
File Explorer - Easily navigate your local files and network connections, and create links with simple drag-and-drop actions.
Clip Library - Store your favorite code snippets in a central location for reuse.
Thumbnail viewer - See all images in a specific folder with the new image thumbnail viewer.
Custom Keyboard Sets - Create sets of custom keyboard shortcuts that insert your favorite tags or other snippets.
Custom Palettes - Create new color palettes by importing existing palettes from Macromedia Fireworks, JASC PaintShop Pro, Adobe PhotoShop (ACT format) and HomeSite.
Harmonious Colors - Create pleasing color schemes for your style sheets and HTML documents with the new harmonious colors feature.
TopStyle Blog - a blog ("web log") is built into TopStyle Pro 3.0, providing daily links of interest to web authors.
CSE HTML Validator Pro Integration - Users of CSE HTML Validator Pro can now access its HTML syntax checking from within TopStyle.

Heewwww! man, thats a mouth full. So whats that saying. It's saying, that if you prefer to edit you html and css manually and generally use WYSIWYG editors for basic layout only; this it the product for you. It even support most scripts.

Where TS shines most is CSS editing. You use a separtate "selector box" to choose your selectors, ID's and elements. You can use a simple integrated interface or, if you like, you can organize you tools by use of a dock system. Those of you that are familiar with Photoshop or Dreamweaver, will feel right at home. And, yes, you can easily switch and save dock layouts by a simple layout selector.

TS is not for the faint of heart, it has a pretty intense learning curve. It's nowhere near that of Photoshop, Dreamweaver and the like though. I found that it took me about full 6 hours of work to where I was comforable and had everything how I wanted.

Also of note is the preview panels. They are up dated after every change, so no switching back and forth between modes to see what the finished product will look like. Also, I mentioned earlier that TS will check you code for errors. Well, first thing you'll want to do is; decide what all will you want it to check for. It'll show compatibility alerts and error for a seemingly endless list of browsers and standards, from Netscape 3.0 to CSS 2.0, not to mention even provide solutions for and reasons behind the errors and warnings. And simplely clicking on the error/warning will automatically select the apropriate tag or ID.

Color management is a snap. With a simple palette selector that link to the same color elements. For instance, if to choose to change the color of my text that have been formatted to "black", simplely select "black" and goto the color chooser and change to another color. Say "Red", now all the elements on my page that are the "black" will be change to "red". Simple as that! No more repetitious cutting and pasting, and no more need for macros or "find and replace". Smooth as butta!

With all that said, I have 2 gripes. First is this package would be absolutely wonderful if it had an FTP client intergrated. That is a sore point to an otherwise top notch program. (They allow a plugin to an external ftp client, but that's no substitute in my opinion.) Second is price, 50 bucks is still a little steep for such a specialized program, no matter how good. I'm so cheap, I'm still using the demo and dread the day when the demo runs out, as I'm morally (and legally as well) obligated to buy it when it expires.

TS is excellent, and I whole-heartedly recommend this product to anyone who doesn't mind editing text, prefer to do so, and want completely clean code for there web pages.

Spam filtering - Napster style!

If you're like me, your probably getting more spam than legit email. I get about 400 spam messages a day through my Yahoo! account alone.

Well, thanks to a simple Outlook plugin, SpamNet, you can enjoy a nearly spamfree inbox. I just installed it today and it has already stopped nearly 300 spam messages from reaching my inbox.

It simply checks your mail as it's being downloaded and checks a network of human edited lists of known spam. If the network says it's spam, it sends it to a separate folder that can be checked periodicly to insure that legitimate emails are not "listed". It's really easy to use, and runs in the back ground. If you still get spam in your inbox, simply click the spam button and it's moves the message to your spam folder and notifies SpamNet of the new spam message, so it will get filtered for other users. Likewise, you can do the reverse if you find a legitimate email in the spam folder.

I'm getting about 90% accuracy, and none of my legit emails have been filtered. It's great! It's only for Outlook users though, but hopefully they will provide support for other clients soon. Get it here.
